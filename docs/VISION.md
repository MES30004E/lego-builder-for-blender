# Vision

LEGO Builder for Blender exists to make LEGO model building feel native inside
Blender while preserving the strengths of dedicated LEGO design tools.

## Why This Project Exists

Blender is excellent for modeling, rendering, animation, and presentation. LEGO
builders should be able to use those strengths without leaving a LEGO-aware
building environment.

## Philosophy

The project should be practical, respectful of LEGO building conventions, and
deeply integrated with Blender rather than bolted on as a simple importer.

## Long-Term Aspirations

- Build LEGO models directly in Blender.
- Browse and place parts from recognized LEGO part libraries.
- Support materials, snapping, caching, export, and scene optimization.
- Enable high-quality rendering and presentation workflows.
- Remain open, maintainable, and approachable for contributors.

## Guiding Principles

- Blender-native first.
- Correctness before convenience.
- Clear architecture before large features.
- Small, reviewable milestones.
- Documentation as project memory.

## Non-Goals

- Replacing Blender's general modeling tools.
- Shipping a proprietary part library.
- Claiming feature support before it exists.
- Optimizing prematurely.
- Implementing large systems without design review.

## Definition of Success

The project succeeds when users can build, inspect, optimize, render, and export
LEGO models in Blender through workflows that feel coherent, reliable, and
maintainable.
